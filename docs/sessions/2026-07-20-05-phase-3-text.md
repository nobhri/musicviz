# Phase 3 Text Retrospective

## Outcome

- Added readable centered title and artist text to the direct FFmpeg pipeline.
- Preserved the verified zoom, synchronized waveform, H.264/AAC output,
  1080 × 1920 dimensions, 30 fps, and matching five-second stream durations.
- Verified the complete render in `ffplay`, sampled frames at 0.5, 2.5, and
  4.5 seconds, and confirmed 150 video frames with `ffprobe`.
- Received user confirmation that `output/phase-3-text.mp4` looked good.
- Completed Phase 3 and advanced the roadmap to Phase 4.

## Decisions

- Use `drawtext=textfile=` with explicit Arial font files rather than embedding
  project-provided title and artist strings in the filtergraph.
- Use fixed white text with a semi-opaque black border: 72 pixels for the title
  and 48 pixels for the artist.
- Use the keg-only Homebrew `ffmpeg-full` executable explicitly because the
  standard Homebrew FFmpeg build does not contain a text-rendering filter.
- Keep text rasterization in FFmpeg for Version 0. Reconsider Python and Pillow
  only if deployment constraints or real-song text layouts demonstrate a need.

## What needed correction

- The installed standard Homebrew FFmpeg build was initially assumed to
  support `drawtext`. Inspecting its filters showed that `drawtext`,
  `subtitles`, and `ass` were all unavailable.
- The fallback options were evaluated before implementation. Generating text
  images with Python and Pillow would have introduced Python media generation
  before the direct FFmpeg pipeline was complete.
- Installing `ffmpeg-full` added more dependencies than the standard formula,
  but its keg-only installation avoids replacing the existing executable and
  keeps the rendering pipeline in FFmpeg.

## Verification

- Rendered `output/phase-3-text.mp4` with
  `/opt/homebrew/opt/ffmpeg-full/bin/ffmpeg`.
- Confirmed H.264 video, AAC audio, 1080 × 1920, `yuv420p`, 30 fps, 150 video
  frames, and five-second video, audio, and container durations.
- Inspected three intermediate frames for text readability, position, zoom,
  waveform changes, and artwork coverage.
- Played the complete output in `ffplay`.
- Ran `git diff --check` for the documentation changes.
